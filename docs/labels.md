# 标签体系说明

NUAAMap 项目的标签体系模仿 [OpenClaw ClawSweeper](https://github.com/openclaw/openclaw)，用于对 Issue 和 Pull Request 进行分类、评估和管理。

## 规模

继承 ClawSweeper 的 Size 体系，字母保留英文。

| 标签 | 含义 |
|------|------|
| `规模：XS` | 极简改动，预计几分钟 |
| `规模：S` | 小改动，预计半天内 |
| `规模：M` | 中等改动，预计 1-3 天 |
| `规模：L` | 大改动，预计 3-7 天 |
| `规模：XL` | 超大改动，预计超过 1 周 |

## 优先级

| 标签 | 含义 |
|------|------|
| `优先级：P0` | 🚨 紧急：安全漏洞、崩溃、核心功能不可用 |
| `优先级：P1` | 🔴 高：用户可见 Bug、功能回归 |
| `优先级：P2` | 🟡 中：常规任务 |
| `优先级：P3` | ⚪ 低：优化、文档、锦上添花 |

## 段位

用 **王者荣耀** 段位评估 Issue 和 PR 的质量。从低到高：

| 标签 | 对应段位 | 含义 |
|------|---------|------|
| `段位：未定级` | — | ❓ 信息不足，尚无法评估 |
| `段位：倔强青铜` | 青铜 III-I | 🥉 初具雏形，仍需补充大量信息 |
| `段位：秩序白银` | 白银 III-I | 🥈 基本清晰，可以正常处理 |
| `段位：荣耀黄金` | 黄金 IV-I | 🥇 描述完整，质量不错 |
| `段位：永恒钻石` | 钻石 V-I | 💎 高质量，细节到位 |
| `段位：至尊星耀` | 星耀 V-I | 🌟 非常出色，堪称范例 |
| `段位：最强王者` | 王者 0+ 星 | 👑 顶级质量，完美无瑕 |

### 评定标准

**Issue 报告质量：** 标题清晰度、复现步骤、截图/日志、环境信息

**PR 完成度：** 改动合理性、代码质量、是否附说明、改动范围是否聚焦

## 类型

| 标签 | 含义 |
|------|------|
| `类型：Bug` | 功能异常，需要修复 |
| `类型：功能请求` | 新功能或改进建议 |
| `类型：文档` | 文档改进或补充 |
| `类型：设计优化` | UI/UX 设计改进 |
| `类型：问题咨询` | 需要讨论或答疑 |

## 小组

| 标签 | 职责范围 |
|------|---------|
| `小组：①手绘地图` | 地图底图、图标、CSS 样式、UI 设计 |
| `小组：②交互功能` | 前端交互、地图点击、弹窗、聊天界面 |
| `小组：③平台搭建` | 项目配置、构建脚本、部署、CI/CD |
| `小组：④数据采集` | QA 知识库、建筑信息数据 |
| `小组：⑤AI智能体` | RAG、聊天 API、智能问答 |
| `小组：⑥坐标标注` | 数据转换脚本、像素坐标标注 |

## 状态

| 标签 | 含义 |
|------|------|
| `状态：需要更多信息` | 等待作者补充信息 |
| `状态：等待确认` | 等待维护者确认 |
| `状态：已确认` | 问题已被确认存在 |
| `状态：进行中` | 正在处理中 |
| `状态：等待审核` | PR 等待审核 |
| `状态：准备合并` | PR 可以合并 |

## 杂项

| 标签 | 含义 |
|------|------|
| `no-stale` | 排除在过时自动关闭之外 |
| `重复` | 此 Issue 或 PR 已存在 |
